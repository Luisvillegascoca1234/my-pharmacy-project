---
title: Ventas POS
description: Flujo operativo de mostrador, pago efectivo, FEFO y comprobante interno.
---

La venta POS representa la operacion de mostrador de la farmacia. Su objetivo es preparar un carrito, cobrar en efectivo, generar un comprobante interno y descontar inventario de manera trazable por FEFO.

## Flujo normal

1. El vendedor busca el producto por nombre, código, código de barras o criterio disponible.
2. Verifica stock apto para venta.
3. Agrega productos al carrito con cantidades enteras.
4. Revisa el total y las advertencias de proximo vencimiento.
5. Registra el monto recibido en efectivo.
6. El sistema calcula el cambio.
7. Al confirmar el cobro, descuenta inventario por FEFO y emite comprobante interno.

## Stock apto para venta

Un producto puede tener stock total, pero no necesariamente stock vendible. Para considerarse apto, el lote debe tener cantidad disponible, estado activo y fecha de vencimiento válida. Este criterio protege a la farmacia de vender productos vencidos o bloqueados.

## Descuento FEFO

Cuando existen varios lotes disponibles, el sistema debe priorizar los de vencimiento más cercano. Si una venta requiere más cantidad que la disponible en un solo lote, puede descontar de varias capas o lotes, siempre conservando la trazabilidad del origen.

## Datos de venta

Una venta debe conservar vendedor, caja, fecha y hora, productos, cantidades, precio unitario, subtotal, total, pago efectivo, cambio y evidencia de los lotes consumidos. En V1 la venta es anonima o de consumidor final y el comprobante interno no reemplaza una factura fiscal.

## Margen comercial

El margen no se calcula solamente con el precio de venta. Debe considerar el costo real del lote o capa descontada. Esta decisión permite saber si una venta fue rentable incluso cuando el mismo producto se recibió con costos diferentes.

## Reglas de negocio

- No debe venderse stock vencido, cancelado o no disponible.
- Las cantidades de POS son enteras.
- La venta nace recien al confirmar el pago.
- El carrito activo no reserva stock.
- El pago V1 es unico y en efectivo.
- El monto recibido debe cubrir el total.
- El comprobante interno y la factura fiscal son conceptos distintos.
- Carritos pendientes y anulacion controlada pertenecen al alcance V1 operativo: el pendiente no reserva stock ni congela precio, y la anulacion conserva evidencia de reversa, reposicion por lote y venta neta de caja mientras la caja asociada siga abierta.

## Resultado esperado

La venta POS debe ser rápida para el mostrador, pero rigurosa para el inventario. Cada producto vendido debe poder rastrearse hasta el lote o capa que fue descontada.
