---
title: Guia de usuarios
description: Como consultar, filtrar y administrar usuarios del sistema.
---

## Ruta

Abrir **Administración > Usuarios**.

## Para que sirve

La pantalla **Usuarios** permite revisar accesos, roles operativos y estado de cuenta. Esta area debe usarse con cuidado porque afecta quién puede operar compras, inventario, ventas y configuraciones.

## Buscar usuarios

1. Entrar a **Administración > Usuarios**.
2. Presionar **Actualizar** si se desea recargar la lista.
3. Escribir en **Buscar usuario** el nombre, correo, rol o estado.
4. Usar el filtro de rol para ver **Todos los roles**, **Administrador**, **Vendedor** o **Superadministrador**.
5. Usar el filtro de estado para ver **Todos los estados**, **Activos**, **Inactivos** o **Bloqueados**.
6. Revisar la tabla de usuarios.

## Crear un usuario

1. Ubicar la seccion **Nuevo usuario**.
2. Completar **Nombre completo**.
3. Completar **Correo electrónico**.
4. Seleccionar **Rol**.
5. Definir el estado inicial si la pantalla lo solicita.
6. Guardar el formulario con el boton de accion visible.

## Editar un usuario

1. Buscar el usuario en la tabla.
2. Presionar **Editar** en su fila.
3. Modificar los datos permitidos.
4. Guardar los cambios.

## Resetear, desactivar o bloquear

1. Ubicar la fila del usuario.
2. Usar **Resetear** cuando se necesite restablecer el acceso segun la regla operativa definida.
3. Usar **Desactivar** para impedir el uso normal sin borrar historial.
4. Usar **Bloquear** cuando exista una condicion de seguridad o control.
5. Confirmar la accion si el sistema muestra un dialogo.

## Controles visibles

- Boton **Actualizar**.
- Campo **Buscar usuario**.
- Filtro por rol.
- Filtro por estado.
- Boton **Editar**.
- Boton **Resetear**.
- Boton **Desactivar**.
- Boton **Bloquear**.
- Formulario **Nuevo usuario**.

## Resultado esperado

La administracion debe poder controlar quién entra al sistema y qué puede hacer. Los cambios de estado no deben borrar la historia operativa del usuario.

## Consejo

Asignar el rol mínimo necesario. Un vendedor no necesita acceso administrativo para registrar ventas, y un admin no debería modificar configuraciones reservadas al superadmin.
