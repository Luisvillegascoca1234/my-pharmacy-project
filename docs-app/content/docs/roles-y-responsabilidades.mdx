---
title: Roles y responsabilidades
description: Responsabilidades operativas de superadmin, admin y vendedor.
---

Los roles delimitan qué acciones puede realizar cada perfil dentro de la farmacia. Esta separación reduce riesgos operativos, protege información sensible y permite que las operaciones críticas queden reservadas a usuarios con responsabilidad administrativa.

## Superadmin

El superadmin representa la máxima autoridad del sistema. Administra usuarios, roles, configuraciones globales, parámetros fiscales y auditoria completa. También puede ejecutar acciones administrativas sensibles, como revisar reportes generales, gestionar catálogos críticos y autorizar operaciones que afecten la trazabilidad.

### Responsabilidades principales

- Gestionar usuarios y estados de acceso.
- Administrar roles y permisos.
- Revisar auditoria completa.
- Configurar criterios globales de operación.
- Gestionar parámetros fiscales cuando se habilite facturacion SIAT.
- Acceder a reportes administrativos y analíticos.

## Admin

El admin se encarga de la operación administrativa diaria de la farmacia. Gestiona productos, unidades, proveedores, compras, inventario, reportes y acciones de control. Su rol está orientado a mantener la consistencia entre abastecimiento, stock disponible y evidencia administrativa.

### Responsabilidades principales

- Registrar y actualizar productos farmacéuticos.
- Gestionar unidades de medida y conversiones.
- Administrar proveedores.
- Crear, recibir y anular compras según reglas.
- Supervisar lotes, vencimientos y stock.
- Autorizar ajustes, devoluciones y anulaciones controladas.
- Revisar reportes operativos.

## Vendedor

El vendedor atiende el flujo de mostrador. Su responsabilidad central es consultar productos, registrar ventas, cobrar y emitir comprobantes cuando corresponda. Este rol debe operar con información suficiente para vender correctamente, pero sin capacidad para modificar reglas críticas del negocio.

### Responsabilidades principales

- Consultar productos y disponibilidad.
- Registrar ventas en el punto de venta.
- Cobrar ventas al contado.
- Consultar alertas operativas básicas.
- Participar en apertura o cierre de caja cuando el proceso lo requiera.

## Restricciones relevantes

El vendedor no debe modificar costos, ajustar inventario manualmente, anular facturas, crear usuarios, cambiar roles ni alterar configuraciones fiscales. Estas restricciones evitan que la misma persona que realiza la venta pueda modificar unilateralmente datos que explican inventario, margen o cumplimiento tributario.

## Resultado esperado

La farmacia debe operar con responsabilidades claras. Cada usuario debe contar con acceso suficiente para cumplir su función, pero sin exponer procesos sensibles a cambios no autorizados.
