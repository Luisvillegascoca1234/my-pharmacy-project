---
title: Consejos operativos
description: Buenas practicas para usar el sistema con datos consistentes.
---

Estos consejos ayudan a reducir errores al operar el sistema. Están pensados para usuarios administrativos, vendedores y responsables de inventario.

## Antes de registrar productos

- Crear primero las unidades de medida y categorias.
- Elegir una unidad base que permita controlar el stock real.
- Usar nombres comerciales claros.
- Registrar nombre generico cuando se trate de medicamentos.
- Activar lote y vencimiento en productos inventariables.

## Antes de registrar compras

- Verificar que el proveedor esté activo.
- Confirmar que los productos estén activos.
- Revisar que cada producto tenga sus unidades configuradas.
- Preparar lote y fecha de vencimiento antes de cargar items.
- Usar notas internas cuando exista una observacion de recepción.

## Al recibir inventario

- Guardar el borrador antes de recibir.
- Revisar proveedor, fecha comercial, items, lote y vencimiento.
- Confirmar que no existan cambios pendientes.
- Usar **Recibir** solo cuando la compra ya represente productos realmente ingresados.

## Al anular o desactivar

- Usar motivos claros y verificables.
- No anular para corregir errores menores si el registro todavia puede editarse.
- Desactivar proveedores o usuarios cuando se quiera impedir nuevas operaciones sin borrar historial.
- Evitar cambios de estado si existen operaciones en curso que dependen de ese registro.

## Para mantener trazabilidad

- No reutilizar codigos internos para productos diferentes.
- No crear categorias duplicadas con nombres parecidos.
- No registrar lotes con variantes visuales innecesarias.
- Revisar reportes y alertas de forma periodica.
- Mantener separados los conceptos de venta, pago, factura y movimiento de inventario.
