## Como leer una guia

Cada guia mantiene la misma estructura:

- **Ruta:** donde se abre la pantalla.
- **Para que sirve:** objetivo operativo.
- **Flujo principal:** pasos que debe seguir el usuario.
- **Botones y controles:** acciones visibles de la interfaz.
- **Resultado esperado:** efecto que debe observarse despues de completar el flujo.

## Regla practica

Antes de presionar una accion sensible, como **Guardar**, **Recibir**, **Anular**, **Bloquear** o **Desactivar**, revisar que los datos visibles correspondan a la operacion real. El sistema conserva historial, por lo que conviene corregir antes de confirmar.
